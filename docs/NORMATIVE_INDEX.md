# Normative vs Informative

**Normative core:** The normative specification consists of:

- **ACTIS_STANDARD_v1.md** — Core standard (formats, verification semantics, report schema).
- **ACTIS_COMPATIBILITY.md** — Verification algorithms (envelope hash, hashing, signatures, actis_status decision tree).

**Informative:** Examples, formulas labeled "(informative)", developer notes, onboarding material (e.g. START_HERE), and governance/IP documents are informative unless explicitly stated otherwise. They do not add or remove binding requirements.
